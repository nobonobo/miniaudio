# miniaudioソース関連メモ

## DataSourceを継承しているもの

| 型                     | 用途                    | よく使うか |
| --------------------- | --------------------- | ----- |
| `ma_decoder`          | 音声ファイルのデコード           | ★★★★★ |
| `ma_audio_buffer`     | メモリ上のPCMデータ           | ★★★★★ |
| `ma_sound`            | 再生中のサウンド              | ★★★★☆ |
| `ma_data_source_base` | 独自DataSource実装用の基底クラス | ★★☆☆☆ |
| ユーザー独自のDataSource     | コールバック等で実装            | ★★☆☆☆ |

## コールバックについて

独自実装を必要としない場合、コールバックの仕組みは無くてもおおむね問題ない

## 用途とオブジェクト

基本のEngineオブジェクトを利用するだけで９割の用途をカバーできる。

- Engine

Engineオブジェクト使っていると以下のオブジェクトが必要であれば生成される

- Sound
- Decoder

反響を入れたりするならグループ化とリバーブノードを利用することになる。
(ここまででほとんどのゲーム音関連実装は十分可能)

- SoundGroup
- Node/NodeGraph

以下のオブジェクトは低レイヤを触りたい場合に必要。

- Device
- ResourceManager
